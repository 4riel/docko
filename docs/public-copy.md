# Public-Facing Copy

## Naming Rationale

`docko` suggests a practical home base for agent work:

- a dock is where workers come and go
- it suggests stable berths rather than disposable checkouts
- it sounds operational, not aspirational

The name supports the core idea of one workspace hub with persistent writable slots.

## Project Pitch

`docko` is a workspace-first session and slot protocol for AI coding agents. It gives teams one stable root for coordination and persistent writable slots, with a first-class Claude Code adapter and guidance-based onboarding for `AGENTS.md` runtimes such as Codex.

For npm, publish the package as [`docko-workspace`](https://www.npmjs.com/package/docko-workspace). Keep the product and CLI name as `docko`.

## Why This Exists

Some teams want more than parallel branches. They want one operational root where shared context lives, slots stay warm, and agent ownership is explicit. `docko` standardizes that model without arguing that worktrees are broken.

## Short Taglines

- Workspace-first slot coordination for AI coding agents.
- One stable root, persistent slots, explicit session ownership.
- A local-first protocol for warm clones and multi-agent work.
- The `docko` workflow, published on npm as [`docko-workspace`](https://www.npmjs.com/package/docko-workspace).
